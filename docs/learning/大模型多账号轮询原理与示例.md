# 大模型多账号轮询原理与示例

本文档用于快速说明：在调用大模型 API 时，为什么要做多账号轮询，以及最简单的实现方式。

## 1. 为什么要做轮询

常见原因：
- 单个账号有调用额度限制（RPM/TPM/日额度）
- 需要更高吞吐量（并发请求更多）
- 某个账号短时失败时希望自动切换到其他账号

简单理解：
- 一个账号像一条车道，轮询就是把请求按顺序分到多条车道。

## 2. 轮询原理（最简版）

核心机制只有两步：

1. 准备账号池（每个账号包含 `base_url + api_key`）
2. 维护一个索引 `idx`，每次请求后执行：

```text
idx = (idx + 1) % 账号总数
```

这样会形成循环：

```text
账号1 -> 账号2 -> 账号3 -> 账号1 -> ...
```

这和遍历数组很像，但更准确叫“循环调度（Round Robin）”。

## 3. 最简可运行示例（Python）

```python
import requests

# 1) 账号池（每个账号一组 url + key）
accounts = [
    {"base_url": "https://api.example.com/v1/chat/completions", "api_key": "KEY_A"},
    {"base_url": "https://api.example.com/v1/chat/completions", "api_key": "KEY_B"},
    {"base_url": "https://api.example.com/v1/chat/completions", "api_key": "KEY_C"},
]

# 2) 轮询索引
current_idx = 0


def get_next_account():
    global current_idx
    acc = accounts[current_idx]
    current_idx = (current_idx + 1) % len(accounts)
    return acc


def call_llm(prompt: str):
    acc = get_next_account()
    headers = {
        "Authorization": f"Bearer {acc['api_key']}",
        "Content-Type": "application/json",
    }
    payload = {
        "model": "your-model-name",
        "messages": [{"role": "user", "content": prompt}],
    }

    resp = requests.post(acc["base_url"], headers=headers, json=payload, timeout=30)
    resp.raise_for_status()
    return resp.json()


if __name__ == "__main__":
    for i in range(6):
        data = call_llm(f"第{i+1}次请求")
        print(f"请求{i+1}完成，响应长度={len(str(data))}")
```

## 4. 需要知道的边界

上面的代码是“教学最简版”，实际生产一般还要加：
- 失败重试（如 429/5xx）
- key 冷却和熔断（短时间内不要重复用故障 key）
- 并发安全（多线程/多进程时索引要加锁或用 Redis）

一句话总结：
- 轮询是起点，稳定可用还需要“限流 + 容错 + 并发控制”。
