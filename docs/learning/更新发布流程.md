# 代码更新发布流程（GitHub + Render）

适用场景：项目已上线，后续修改代码后重新发布。

## 1. 本地更新后先自测

在本地确认：
- 页面能正常打开
- 关键功能可用（上传、对账、导出）

建议至少跑一次：
- 上传供应商文件
- 上传入库单
- 执行自动对账

---

## 2. 提交并推送到 GitHub

在项目目录执行：

```bash
cd /d D:\Reconciliation
git status
git add .
git commit -m "feat/fix: 说明本次更新"
git push
```

说明：
- 如果是首次推送过，后续通常只要 `git push`（远程已关联）
- `git status` 可确认哪些文件会被提交

---

## 3. Render 更新方式

### 情况A：开启 Auto Deploy（推荐）
你 `git push` 后，Render 会自动开始新一轮部署。

检查路径：
- Render 控制台 -> 你的服务 -> Deploys
- 看最新 Deploy 状态是否 `Live`

### 情况B：未开启 Auto Deploy
手动触发：
- Render 控制台 -> 你的服务 -> Manual Deploy
- 选择 `Deploy latest commit`

---

## 4. 发布后验收

1. 打开线上 URL
2. 强制刷新（Ctrl+F5）避免旧缓存
3. 验证本次改动是否生效
4. 核心功能回归测试（上传/对账/导出）

---

## 5. 常见问题

### 5.1 push 被拒绝（non-fast-forward）
先拉取再推送：

```bash
git pull --rebase origin main
git push
```

### 5.2 Render 构建失败
看 Build Logs：
- 缺依赖：补到 `requirements.txt` 后重新 push
- 启动失败：确认 Start Command 为：

```bash
gunicorn app:app --bind 0.0.0.0:$PORT
```

### 5.3 页面看起来没更新
- 浏览器缓存导致，先 `Ctrl+F5`
- 检查 Render 最新部署是否是刚刚那次 commit

---

## 6. 日常最简步骤（速记）

```bash
git add .
git commit -m "update"
git push
```

然后去 Render 看最新 Deploy 是否 `Live`。
