# GitHub Push + Render 部署通用流程

适用项目：Python Flask（含 `app.py`、`requirements.txt`）

## 1. 上线前检查（本地）

### 必备文件
- `app.py`
- `requirements.txt`
- `templates/`
- `static/`

### 生产启动要求
`app.py` 末尾建议：

```python
if __name__ == "__main__":
    port = int(os.environ.get("PORT", 5000))
    app.run(debug=False, host="0.0.0.0", port=port)
```

### 依赖要求
`requirements.txt` 需包含 `gunicorn`：

```txt
gunicorn>=21.2.0
```

### 忽略文件（建议）
创建 `.gitignore`，至少包含：

```gitignore
.venv/
__pycache__/
.ruff_cache/
uploads/
*.pyc
```

---

## 2. Git 推送 GitHub 固定流程

在项目目录执行：

```bash
cd /d D:\Reconciliation
git init
git add .
git commit -m "deploy: prepare production"
git branch -M main
git remote add origin https://github.com/<你的用户名>/<仓库名>.git
git push -u origin main
```

如果提示远程已存在：

```bash
git remote set-url origin https://github.com/<你的用户名>/<仓库名>.git
```

如果首次提交报身份错误（Author identity unknown）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的GitHub邮箱"
```

---

## 3. Render 部署固定参数

Render -> New -> Web Service -> 选择 GitHub 仓库 -> 填：

- Branch: `main`
- Runtime: `Python`
- Build Command:

```bash
pip install -r requirements.txt
```

- Start Command:

```bash
gunicorn app:app --bind 0.0.0.0:$PORT
```

- Health Check Path: `/`（建议）

---

## 4. 部署后验收清单

1. 页面可打开（非 502/503）
2. 首页 `/` 返回正常
3. 上传供应商文件正常
4. 上传入库单正常
5. 点击自动对账能返回结果
6. 导出结果正常下载

---

## 5. 常见坑与解决

### 坑1：`Author identity unknown`
原因：git 未配置用户名邮箱。
解决：执行第2节里的 `git config --global ...`

### 坑2：`gh auth login` 网络超时
原因：本机到 GitHub 443 不通（公司网/校园网常见）。
解决：
- 临时换网络（手机热点）
- 或配置代理
- 或直接用 `git push`（不用 gh）

### 坑3：Render 构建成功但一直 Deploying
排查顺序：
1. 看 Runtime Logs（不是 Build Logs）
2. 确认 Start Command 是 `gunicorn app:app --bind 0.0.0.0:$PORT`
3. Health Check 路径设 `/`
4. 手动 Redeploy 一次

### 坑4：`gunicorn: command not found`
原因：`requirements.txt` 没有 gunicorn。
解决：补充依赖后重新 push。

### 坑5：上传文件重启后丢失
原因：云平台磁盘多为临时盘。
解决：如需长期保存，接对象存储（S3/OSS/COS）。

### 坑6：批处理脚本乱码/命令异常
原因：`.bat` 编码不一致导致。
解决：
- 脚本尽量用 ASCII 文案
- 保存为 ANSI/UTF-8（无 BOM）并测试

---

## 6. 下次复用最简版本（速查）

```bash
git add .
git commit -m "update"
git push
```

Render 会自动拉取并重新部署。
